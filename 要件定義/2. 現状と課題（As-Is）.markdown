# 1. はじめに

## 1-1. 本書の目的

本書は、日本電通株式会社様向けの「類似インシデント予防策提示システム（以下、本システム）」において、中間システム（AWS上）が利用するデータ永続化層の設計を定義することを目的とします。

本文書で定義するデータストアのうち、DynamoDB は主にテナント情報、ユーザー認証情報、および入力画面の選択肢となる階層型工事マスタデータの管理に利用されます。

## 1-2. スコープ

### **対象範囲**

- **本システムが利用する DynamoDB のテーブル設計**
  - キースキーマ（パーティションキー、ソートキー）
  - 属性定義
  - インデックス設計（GSI）
  - （管理対象：テナント情報、ユーザー情報、階層型工事マスタデータ）

### **対象外の範囲**

- AWSインフラストラクチャ（VPC、Lambda、API Gateway等）の構築手順や設定値
- VideoQuestor (RAGモデル) への連携仕様詳細および、RAGモデルが参照するインシデント報告資料（PDFやDOC等）の管理
- フロントエンド（UI）の画面設計およびHTML/CSS/JavaScriptの実装
- 認証（ログイン）処理のシーケンスやロジック

## 1-3. 用語定義

| 用語 | フルスペル                     | 説明                                             |

| RAG  | Retrieval-Augmented Generation | 検索拡張生成モデル                               |
